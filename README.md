# 📰 ニュースダッシュボード 公開手順書

大企業の新規事業ニュースと生成AI情報を毎朝7時に自動収集して表示するダッシュボードです。
この手順書のとおりに操作すれば、プログラミングの知識なしで公開できます。

## このフォルダの中身

| ファイル | 役割 |
|---|---|
| `index.html` | ダッシュボード画面（見る用のページ） |
| `data/news.json` | 集めたニュースの保存ファイル（自動更新される） |
| `scripts/collect_news.py` | ニュース収集プログラム |
| `.github/workflows/daily.yml` | 「毎朝7時に収集して」という予約設定 |

## 公開手順（所要15分・すべてブラウザ上の操作）

### ステップ1：置き場所（リポジトリ）を作る

1. [github.com](https://github.com) にログイン
2. 右上の「＋」→「**New repository**」をクリック
3. 「Repository name」に `news-dashboard` と入力
4. 「**Public**」を選択（Pagesで公開するために必要。ページは誰でも見られる状態になります）
5. ほかは何も変えず「**Create repository**」をクリック

### ステップ2：ファイルをアップロードする

1. 作成直後の画面にある「**uploading an existing file**」というリンクをクリック
2. パソコンでこの `news-dashboard` フォルダを開き、**中身をすべて選択**（`index.html`、`data`、`scripts`、`.github` の4つ。この手順書 README.md も一緒で構いません）して、ブラウザの点線の枠にドラッグ＆ドロップ
3. ファイル一覧に `.github/workflows/daily.yml` が含まれていることを確認
4. 下の「**Commit changes**」ボタンをクリック

> ⚠️ もし `.github` フォルダがドラッグできない場合：アップロード後、リポジトリの「Add file」→「Create new file」を選び、ファイル名欄に `.github/workflows/daily.yml` と入力（スラッシュを打つと自動でフォルダになります）。手元の `daily.yml` をメモ帳で開いて中身をコピー＆貼り付けし、「Commit changes」してください。

### ステップ3：ページを公開する（GitHub Pages）

1. リポジトリ上部の「**Settings**」タブ →左メニューの「**Pages**」
2. 「Branch」のプルダウンで「**main**」を選び「**Save**」
3. 1〜2分待つと、ページ上部に公開URL（`https://あなたのユーザー名.github.io/news-dashboard/`）が表示されます。これがダッシュボードのアドレスです

### ステップ4：収集を一度動かして動作確認

1. リポジトリ上部の「**Actions**」タブを開く（有効化を求められたらボタンを押して有効化）
2. 左の「**ニュース自動収集**」→右側の「**Run workflow**」→ 緑の「**Run workflow**」ボタン
3. 1〜2分で完了（緑のチェックマークが付きます）
4. 公開URLを開き、最新ニュースが並んでいれば完成です 🎉

以降は毎朝7時ごろに自動更新されます。チームにはURLを共有するだけでOK。スマホでも見られます。

## カスタマイズ方法

`scripts/collect_news.py` の冒頭「設定エリア」を編集します（GitHub上でファイルを開き、鉛筆アイコンで編集→Commit changes）。

- **メディアを追加/削除**：`FEEDS` の行を追加・削除
- **拾うキーワードを調整**：`AI_KEYWORDS`（生成AIジャンル）、`BIZ_KEYWORDS`（新規事業ジャンル）に言葉を追加。例：自社の注目業界「物流」「金融」など
- **保存日数の変更**：`DAYS_TO_KEEP = 14` の数字を変更

## 知っておくと安心な注意点

- **実行時刻は多少ずれます**：GitHubの混雑状況により、朝7時ちょうどではなく数分〜数十分遅れることがあります（GitHubの仕様）
- **一部メディアは未検証です**：CNET Japan と ZDNET Japan は、作成時の環境の制約で配信テストができませんでした。もし取得できなくても自動でスキップされ、他のメディアには影響しません（それ以外の11メディアは配信中であることを確認済み）
- **手元でダブルクリックしても記事は表示されません**：ブラウザの安全機能によるもので、故障ではありません。公開URLでご覧ください
- **最初に入っているニュースはテスト用の実データです**（ASCII.jp・東洋経済・OpenAIの一部のみ）。ステップ4を実行すると全メディアの最新データに置き換わります
- **長期間放置した場合**：リポジトリに60日間まったく動きがないと自動実行が停止することがあります（GitHubの仕様）。その場合はActionsタブを開くと表示される「再有効化」ボタンを押せば復活します
